1. JAR checkInput_Tele2_v2.jar служит для обработки и корректировки входных файлов на соответствие DP, полученных от заказчика (Tele2).

Параметры для запуска JAR:
 1 параметр - входной файл или директория с входными файлами
 2 параметр - файл-шаблон (.tpl) : ProfileTemplate_Tele2.tpl

Внимательно: при запуске, например, из Double Commander jar файл и оба параметра (при этом надо брать полный путь и для jar и для всех параметров через ctrl+Shift +C) надо экранировать кавычками.
То есть во всех путях добавлять кавычки слева и справа! (иначе программа не запуститься).

Пример запуска из Double Commander:
"z:\GSM_Customers_new\TELE_2\Tele2_2020\eUICC\Обработка входных данных\tools\Tools_simplified_template\checkInput_Tele2.jar" "c:\Users\Ivan\IdeaProjects\Tele2_checkAndCorrectInput_v2\TTTTTTTele2" "z:\GSM_Customers_new\TELE_2\Tele2_2020\eUICC\Обработка входных данных\tools\Tools_simplified_template\ProfileTemplate_Tele2.tpl"

Снял ограничение работы программы: кроме входных файлов (расширение .inp или .txt) в директории
не должно быть других каких-либо файлов - иначе директория с обработанными файлами to_ECP - будет пустой (ограничение было в 2020 году).

Теперь файлы внутри директории, отличающиеся по формату от входных файлов могут также находиться внутри
директории для обработки вместе с входными файлами, например файл count.txt, служащий для подсчёта суммарного количества профилей (действует с 2021 года).



Если на входе только 1 входной файл, то на выходе получаются файлы с _back и без _back.

Берём только тот входной файл, который без _back - он корректный.
Также создаётся файл .log с именем этого файла в текущей директории.

Если на входе директория с файлами, то создаётся внутри этой директории директория to_ECP,
в которую кладутся уже обработанные файлы, которые соответствуют в результате обработке DP (шаблону).

В этом случае брать файлы из директории to_ECP.

Также в этом случае создаётся директория fail, в которой создаётся файл log: not correct files.log,
в котором указываются имена тех входных файлов, которые были изначально некорректные
(Был изменён входной файл: имя файла).

Смысл в том, что в файле-шаблоне во всех строках после : можно ставить только %, и всё равно будет происходить замена.
 Главное, чтобы все ключи в файле-шаблоне были и во входном файле (тогда будет корректная замена строк для сравнения).
Для этого создан отдельный метод.
